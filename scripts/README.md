# 占位图片生成器使用说明

## 🎯 目的

为羊了个羊游戏生成占位卡牌图片，用于开发和测试。

## 📋 使用步骤

### 方法1：使用HTML生成器（推荐）

1. 在浏览器中打开 `generatePlaceholderImages.html`
   ```bash
   open scripts/generatePlaceholderImages.html
   ```

2. 页面会自动生成6张卡牌占位图片预览

3. 点击 **"📦 下载全部图片 (ZIP)"** 按钮下载压缩包

4. 解压ZIP文件，将6张图片放入项目的 `src/images/cards/` 目录：
   ```
   src/images/cards/
   ├── card_01.png  (羊 🐑)
   ├── card_02.png  (牛 🐄)
   ├── card_03.png  (猪 🐷)
   ├── card_04.png  (狗 🐶)
   ├── card_05.png  (猫 🐱)
   └── card_06.png  (兔 🐰)
   ```

5. 启动游戏测试：
   ```bash
   npm run dev
   ```

### 方法2：单张下载

- 在HTML页面中，右键点击单张图片，选择"图片另存为"
- 或者点击图片下方的"下载"按钮

## 🎨 图片规格

- **尺寸**: 256x256 px
- **格式**: PNG
- **特点**:
  - 渐变背景（每种动物不同颜色）
  - 白色圆形边框
  - 大号emoji居中显示
  - 底部标注动物名称

## 🔄 替换为正式图片

将来需要替换为正式美术素材时：

1. 准备6张PNG图片（建议256x256或更高）
2. 命名为 `card_01.png` ~ `card_06.png`
3. 放入 `src/images/cards/` 目录覆盖占位图片
4. 如需调整缩放，修改 `src/config/gameConfig.js` 中的 `cardImageScale` 配置

## ⚠️ 注意事项

- 文件命名必须严格按照 `card_01.png` 格式，数字补齐两位
- 图片必须是PNG格式
- 建议使用透明背景，方便后期调整
- 如果图片加载失败，游戏会自动降级使用emoji显示

## 🐛 故障排除

**问题**: 游戏中看不到图片，只显示emoji

**可能原因**:
1. 图片未放入正确目录 (`src/images/cards/`)
2. 图片命名错误
3. 图片格式不是PNG

**解决方法**:
1. 检查控制台是否有加载失败警告
2. 确认图片路径和命名正确
3. 重新启动开发服务器
